이 디렉터리의 문서는 원본 PDF를 구조화하고 확정된 제품 결정을 반영한 현재 구현 계약이다.

## 구현 불변 조건

- 데이트 평가는 장소별 하트와 한 줄 코멘트만 사용한다. 평점·만족도 필드는 없다.
- 장소 식별자는 TourAPI `contentId`를 API와 DB에서 동일하게 사용한다.
- 코스는 DB에 존재하는 장소 2~4개로 구성한다.
- 데이트 성향은 프로필에 저장하지 않고 코스를 만들 때마다 입력한다.
- 현재 장소 완료는 순서대로 처리하며 재요청은 멱등하다.
- 마지막 장소 완료 후에도 한 줄 코멘트로 종료하기 전까지 코스는 `ACTIVE`다.
- 인기 랭킹과 데이트 마스터는 게시글 좋아요 수를 기준으로 한다.
- 삭제 게시글 재발행은 새 행을 만들지 않고 기존 행과 좋아요를 유지한 채 복원한다.
- 랭킹 복사와 과거 재진행은 장소 하트와 진행 상태를 초기화한다.
- 날씨 공급자 장애는 코스 생성 전체 실패로 이어지지 않는다.
- 축제·공연 추천과 현재 데이트 취소 기능은 제공하지 않는다.

## 교차 참조 규칙

- 기능을 구현할 때 기능 ID에서 시작해 대응 API와 DB 트랜잭션을 함께 확인한다.
- API DTO 필드는 `camelCase`, DB 컬럼은 `snake_case`를 사용한다.
- Markdown 명세와 원본 PDF가 다르면 Markdown 명세가 현재 계약이다.
