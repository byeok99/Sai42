## 1. 제품 범위

사이42는 익명 커플 프로필로 입장해 매 데이트마다 조건을 새로 입력하고, 대전 공공데이터 장소를 기반으로 AI 데이트 코스를 만든 뒤 현재 데이트, 랭킹보드, 추억을 관리하는 서비스다.

핵심 흐름은 다음과 같다.

1. 닉네임과 숫자 4자리 비밀번호로 익명 프로필을 등록하거나 기존 기록을 불러온다.
2. 날짜, 시간대, 지역, 공간, 분위기, 활동, 밀도, 이동수단을 매번 입력한다.
3. 날씨와 공공데이터 장소를 조회해 AI가 2~4개 장소로 코스를 만든다.
4. 자연어로 초안을 조정한 뒤 현재 데이트로 확정한다.
5. 데이트 중 장소별 하트를 기록하고, 종료 시 한 줄 코멘트를 남긴다.
6. 완료 코스는 추억에 보존되고 랭킹보드에 자동 공개된다.

|---|---|---|---|

| BOT-26 | 빠른 수정 | 카페 변경, 야경 추가, 동선 축소, 실내 강화 기능을 제공한다. | 필수 |
| BOT-27 | 채팅 기록 | 사용자와 AI 대화를 누적 표시한다. | 필수 |
| BOT-28 | 코스 재생성 | 같은 조건으로 다른 코스를 생성한다. | 선택 |
| BOT-29 | 코스 결정 | 코스를 현재 데이트로 확정한다. | 필수 |

확정된 코스의 제목, 장소, 순서, 시간은 수정할 수 없다.

빠른 수정과 재생성은 코스 조정 API의 `CourseEditAction`으로 전달한다.

| 기능 | Action |
|---|---|
| 카페 변경 | `CHANGE_CAFE` |
| 야경 추가 | `ADD_NIGHT_VIEW` |
| 동선 축소 | `REDUCE_ROUTE` |
| 실내 강화 | `INCREASE_INDOOR` |
| 같은 조건 재생성 | `REGENERATE` |

최초 생성, 빠른 수정, 자연어 수정, 재생성 이후에도 코스는 DB에 존재하는 장소 2~4개를 유지한다. 전체 예상 소요시간은 첫 장소 시작부터 마지막 장소의 예상 종료까지 계산한다.

## 4. 현재 데이트

| ID | 기능 | 설명 | 우선순위 |
|---|---|---|---|
| CUR-01 | 현재 코스 조회 | 프로필의 `ACTIVE` 코스를 조회한다. | 필수 |
| CUR-02 | 진행률 표시 | 현재 장소와 전체 진행률을 표시한다. | 필수 |
| CUR-03 | 지도·카드 연동 | 마커와 장소 카드를 연결한다. | 필수 |
| CUR-04 | 다음 장소 | 현재 장소를 완료하고 다음 장소로 이동한다. | 필수 |
| CUR-05 | 장소 하트 등록 | 현재 코스의 장소에 하트를 누른다. | 필수 |
| CUR-06 | 장소 하트 취소 | 하트를 다시 눌러 취소한다. | 필수 |
| CUR-07 | 데이트 종료 | 코스를 `COMPLETED` 상태로 변경한다. | 필수 |
| CUR-08 | 한 줄 코멘트 | 종료 시 한 줄 코멘트를 입력한다. | 필수 |
| CUR-09 | 랭킹 자동 공개 | 종료와 동시에 게시글을 자동 생성한다. | 필수 |

현재 데이트 취소 기능은 없다. 종료 시 전체 별점이나 만족도 점수를 입력하지 않는다.

장소는 방문 순서대로만 완료할 수 있다. 이미 완료된 장소를 다시 완료 요청해도 다음 장소로 두 번 이동하지 않는다. 마지막 장소를 완료하면 진행률은 100%가 되지만, 한 줄 코멘트로 데이트를 종료할 때까지 코스 상태는 `ACTIVE`다.

## 7. 공통 정책

- 축제 관련 기능은 사용하지 않는다.
- 공공데이터 장소 구조는 현재 설계를 유지한다.
- 데이트 성향은 프로필에 저장하지 않고 매번 새로 입력한다.
- 장소별 하트는 현재 데이트에서만 변경할 수 있다.
- 게시글 좋아요와 장소 하트는 서로 다른 데이터다.
- 데이트 종료 평가는 장소별 하트와 한 줄 코멘트만 사용한다.
- 현재 데이트 취소는 제공하지 않는다.
- 확정 및 완료된 코스는 수정할 수 없다.
- 랭킹이나 추억에서 코스를 다시 진행할 때 과거 장소 하트와 방문 완료 상태는 복사하지 않는다.
- 삭제한 게시글을 다시 공개할 때는 기존 게시글 행과 좋아요를 유지하고 공개 시각과 한 줄 코멘트를 갱신한다.

## 8. 기능 수용 기준 요약

- 필수 기능 58개, 선택 기능 2개로 총 60개 기능 ID를 유지한다.
- 코스 생성 입력은 매 요청에 완전한 조건 세트를 포함해야 한다.
- AI는 SQLite에 적재된 추천 가능 장소만 코스에 넣어야 한다.
- 코스는 항상 장소 2~4개와 전체 예상 소요시간을 제공해야 한다.
- 한 프로필은 동시에 하나의 `ACTIVE` 코스만 가질 수 있다.
- 현재 장소 완료는 순차적이고 멱등하게 처리해야 한다.
- 데이트 종료와 랭킹보드 자동 공개는 하나의 원자적 처리여야 한다.
- 평점 필드나 만족도 점수를 요구하거나 저장하면 안 된다.
- 인기 랭킹과 데이트 마스터는 게시글 좋아요 수를 기준으로 계산해야 한다.
